### 2026-06-11 — Auth method & flow strategy
- **Auth methods (target):** Passkey-first + magic link + password fallback. ⚠️ Passkeys require Keycloak 26.0.7 → 26.4+ upgrade (native). Magic link + password ship on current version. Design assumes the upgrade is on the roadmap; passkey UI gracefully degrades if not yet available.
- **KEY INSIGHT — most users use Gmail/personal email**, not corporate domains. Therefore:
  - **"Continue with Google" is the PRIMARY, most-prominent action** on the login/signup screen.
  - Email-*domain* SSO inference is NOT the lead (public domains like gmail.com can't map to a tenant IdP).
  - **Enterprise SSO = secondary path**: auto-detected only when a *corporate* email domain matches a configured IdP; manual fallback retained. Replaces today's "type your SSO name" alias screen.
- **Signup:** Minimal fields (email + name) + **deferred email verification** — land user in-product immediately, verify async. (3-field forms ~25% conversion vs ~15% for 6+.)
- Implication for login screen hierarchy: ~~[1] Continue with Google~~ → see REVISION below.

### 2026-06-11 — REVISION: no social sign-in
- **User decision: NO social sign-in (no "Continue with Google") for now.**
- Revised login hierarchy: **[1] Email field → Continue (PRIMARY)** → routes to passkey (if registered) / magic link / password · **[2] "Sign in with SSO" (secondary)** for corporate-domain users. No social buttons.
- Most users still use Gmail addresses, but they authenticate via email + passwordless (passkey/magic-link/password), not Google OAuth.
- Passkey-first + magic link + password decision STANDS (passkeys need KC 26.4+ upgrade).

### 2026-06-11 — Key journeys (confirmed, revised for no-social)
1. **Priya — self-serve founder (new).** Enters email → Continue → new account: minimal fields (name), deferred verification, magic link / set passkey → lands in → single inline "name your workspace" → **CLIMAX: inside her new workspace in under a minute, no password wall, no verification gate.**
2. **Sam — invited teammate.** Invite email ("Priya invited you to join Priya's Studio") → email pre-filled → authenticates (magic link/passkey) → invited to exactly one workspace → **auto-accepted, dropped straight in** → **CLIMAX: email to inside the workspace, zero accept/reject screens.**
3. **Dana — consultant in 5 client workspaces.** Returning → passkey (Face ID) → belongs to many → tenant-picker with last-used highlighted + type-to-filter → **CLIMAX: jumps to the right client in two taps, no long-list scan.**
